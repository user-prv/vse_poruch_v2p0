# Міграція/синхронізація даних

## Підхід

1. Зняти snapshot legacy БД.
2. Експортувати довідники та критичні сутності у staging таблиці.
3. Виконати валідацію кількості/контрольних сум.
4. Ідемпотентно завантажити дані у target схему.

## Rollback

- Тримати резервний backup перед cutover.
- Використовувати feature flag на рівні маршрутизації трафіку.
- У разі інциденту:
  1. Перемкнути трафік на legacy.
  2. Відкотити зміни схеми/даних із backup.
  3. Провести postmortem.
