# Інструкція: вхід в адмінку та створення категорій

## 1) Як зайти в адмінку (роль адміністратора)

### Передумови
- Запущені frontend + backend (через `infra/docker-compose.yml` gateway публікує застосунок на `http://localhost:5173`).
- Користувач має роль `admin`.

### Який email отримує роль admin автоматично
У backend є allowlist email-адрес адміністраторів. Зараз це:
- `6353577@gmail.com`

Під час реєстрації/логіну з цією адресою backend встановлює роль `admin`.

### Кроки входу
1. Відкрийте `http://localhost:5173/login`.
2. Увійдіть під email адміністратора (`6353577@gmail.com`).
3. Після логіну відкрийте `http://localhost:5173/admin` (або пункт **Admin** у верхній навігації).
4. Якщо роль не `admin`, RouteGuard перенаправить на `/dashboard`.

## 2) Як створювати категорії

### Варіант A: через UI адмінки
1. Перейдіть у `http://localhost:5173/admin`.
2. В секції **Категорії** знайдіть поле **Нова категорія**.
3. Введіть назву категорії.
4. Натисніть **Додати категорію**.
5. Після успіху категорія з'явиться у таблиці категорій.

> Примітка: кнопка створення активна лише для користувача з роллю `admin`.

### Варіант B: через API
Виконайте POST-запит:

```bash
curl -X POST http://localhost:8080/api/v1/categories \
  -H 'Content-Type: application/json' \
  -d '{"name":"Електроніка"}'
```

Мінімальне payload-поле: `name`.

## 3) Швидка перевірка, що все спрацювало
- В UI: у секції **Категорії** видно новий рядок.
- Через API:

```bash
curl http://localhost:8080/api/v1/categories
```

У відповіді має бути нова категорія.
