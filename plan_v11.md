# План доробки проєкту (v11)

> Формат: чекліст із можливістю відмічати виконані пункти.

## 1) Підготовка тестових даних
- [ ] Створити набір тестових категорій (мінімум 8–10 штук із різною вкладеністю/типами).
- [ ] Створити набір тестових оголошень (мінімум 30–50) з різними станами, цінами, локаціями та авторами.
- [ ] Передбачити зв’язки: кожне оголошення прив’язане до валідної категорії.
- [ ] Додати seed-скрипт/інструкцію запуску для відтворення даних у будь-якому середовищі.
- [ ] Перевірити, що дані коректно відображаються в UI та API.

## 2) Виправлення логіки додавання та редагування оголошень
- [ ] Проаналізувати поточний flow створення/редагування (валідація, збереження, оновлення полів, медіа).
- [ ] Виправити помилки у валідації форми (обов’язкові поля, формат ціни, координати, категорія).
- [ ] Виправити збереження та оновлення оголошення без втрати існуючих даних.
- [ ] Забезпечити коректну обробку чернеток/неповних змін (за потреби).
- [ ] Покрити сценарії unit/integration тестами.

## 3) Верифікація оголошень адміністратором
- [ ] Додати/уточнити статуси оголошень: `draft`, `pending_verification`, `active`, `rejected`, `archived` (або еквівалент).
- [ ] Після створення/суттєвого редагування переводити оголошення у `pending_verification`.
- [ ] Реалізувати адмін-дії: підтвердити (`active`) / відхилити (`rejected`) з причиною.
- [ ] Додати журнал/історію зміни статусів (хто, коли, що змінив).
- [ ] Налаштувати нотифікації автору про результат верифікації.

## 4) Відображення всіх оголошень у кабінеті користувача
- [ ] У кабінеті показувати всі оголошення користувача незалежно від статусу.
- [ ] Для кожного оголошення відображати бейдж статусу (наприклад: «На перевірці», «Активне», «Відхилене»).
- [ ] Додати фільтри в кабінеті за статусом та датою оновлення.
- [ ] Для `rejected` показувати причину відхилення.
- [ ] Перевірити пагінацію/продуктивність для великої кількості оголошень.

## 5) Уніфікація стилів усього проєкту зі стилями головної сторінки
- [ ] Провести UI-аудит відмінностей (типографіка, кольори, кнопки, інпути, відступи, карточки).
- [ ] Винести токени дизайну (кольори, радіуси, тіні, розміри шрифтів) у спільну систему стилів.
- [ ] Уніфікувати базові компоненти (кнопки, форми, модальні, картки, таблиці).
- [ ] Пройтися по ключових сторінках та усунути візуальні розбіжності.
- [ ] Провести регресію UI (desktop + mobile).

## 6) Прибрати нефункціональний пін з карти на головній
- [ ] Знайти джерело рендера нефункціонального піна.
- [ ] Видалити або умовно не рендерити пін, якщо він не має валідних даних.
- [ ] Переконатися, що інші піни/кластеризація працюють без регресій.
- [ ] Оновити тести/скріншоти (за наявності).

## 7) Виправити сторінку категорії
- [ ] Забезпечити одночасне відображення списку товарів і карти на сторінці категорії.
- [ ] Додати сортування товарів за ціною (ASC/DESC).
- [ ] Додати сортування за відстанню (за наявності координат користувача/товару).
- [ ] Коректно обробити edge cases: відсутні координати, порожня категорія, помилки геолокації.
- [ ] Перевірити синхронізацію між картою та списком (виділення/hover/клік).

## 8) Оновити головну сторінку: прибрати сортування за статусом
- [ ] Прибрати UI-контрол сортування/фільтрації за статусом оголошення з головної.
- [ ] На рівні API/бекенду обмежити вибірку для користувачів тільки оголошеннями зі статусом `active`.
- [ ] Перевірити, що неактивні/неверифіковані оголошення не потрапляють у публічну видачу.
- [ ] Переконатися, що для адмінів (за потреби) логіка не зламана.

## 9) Повне тестування функціоналу
- [ ] Скласти тест-план (smoke + regression + рольові сценарії: користувач/адмін).
- [ ] Прогнати unit, integration та e2e тести для критичних сценаріїв.
- [ ] Перевірити основні користувацькі потоки: реєстрація, створення оголошення, верифікація, публічний перегляд.
- [ ] Перевірити фільтрацію/сортування (ціна, відстань, статуси) та роботу карт.
- [ ] Зафіксувати баги, пріоритезувати та повторно прогнати тести після виправлень.

## Критерії готовності (Definition of Done)
- [ ] Функціонал створення/редагування оголошень працює стабільно.
- [ ] Верифікація адміністратором реалізована та прозора для користувача.
- [ ] У кабінеті видно всі оголошення з актуальними статусами.
- [ ] Головна та сторінка категорії відповідають вимогам і не мають критичних дефектів.
- [ ] Тести пройдено, критичні баги закрито.
